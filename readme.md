# 设计方案文档

## 一、工具使用纪实

1. 使用豆包大模型进行生成

2. 提示词设计思路

  根据项目要求，准确描述出项目所有要求的内容，初步生成后打开网页文件逐项测试，根据测试出的问题一一修改，每次修改后再次测试，发现问题再次修改，直到所有内容均测试完好
  测试完成后，发现选项无法点击，让AI修改，文件不适配手机端，描述手机端的要求，如在手机端显示下拉列表，在电脑端平铺显示。

3. 文件的转换
  AI生成的文件默认为markdown格式，而本次项目要去使用HTML格式，需进行转换。
  我使用pandoc工具进行转换，配置好pandoc环境及其环境变量，调用pandoc命令行进行转换吗，为保持样式一致，转换后需复制css部分到新网页。

4. 编程工具的使用

	使用VSCode作为编辑器，安装live server插件实现编辑后自动更新以方便测试。

5. 部署
  将在线图片转为本地图片以防止图片过期
  将网页通过Git上传GitHub Pages 以实现随时访问而不需要保存文件。域名配置好A记录与CNAME记录，申请Let's Encrypt数字证书以实现https访问。

  

## 二、网页规划蓝图

​	分为足迹地图，旅行体验，梦想目的地三个部分。将这三个部分分别写不同旅行的经历。
将每一次经历拆分成独立的文件。并将标题设置为跳转的入口，点击标题跳转到相应的经历。为方便移动端访问，增加侧边栏。侧边栏可以在移动端快速跳转到相应的地区。

| 网站部分   | 内容                                                         |
| ---------- | ------------------------------------------------------------ |
| 足迹地图   | 分为已打卡未打卡两部分，已经去过的为已打卡，未去过的为未打卡部分，分别写了不同目的地的路线规划。 |
| 旅行体验   | 书写各地旅行的体验。                                         |
| 梦想目的地 | 写自己未来旅游的目的地与想做的事情。                         |
| 备案号     | 根据相关法规的要求添加ICP备案号并链接到工信部网站。公安备案号链接到公安部网站。 |
| 版权所有   | 最后添加版权所有，学号、姓名等信息。                         |

​	内容为描述相关要求让AI生成，图片需单独生成以免出错

## 三、创作反思总结
​	在制作过程中，我向AI提供了一个网站的示例。但是AI就跟着网站的示例改变了最终生成网站的内容。
解决方法是：手动描述相应的网站样式，避免让AI同步现有的网站样式。否则AI将会混淆相关意思，造成网站内容与生成的内容大相径庭。

​	另外多次询问问题可能会导致代码混乱，须重开会话或过一段时间再试。
